Introduction to the project:
The project that our group chose to do is a video game, and a website about the game. The game should be the major part of the assignment, and the website should be the minor part of the assignment.

The game:
The game that we designed is similar to an online game called “Soul knight”, the game is a shooting game, where the winning condition is to pass all levels in the game. Things that we are proud of:
Function implemented:
- In the game, Ryan coded methods for spawning rooms and levels, making the game having a bit of randomness, not having the exact same map and room each time when a player is playing the game.
- In the game, Peter coded methods for updating the position of the enemy and the enemy’s bullet, this method involved calculating the angle between the player and the enemy, and adjusting the x value and the y value for the enemy and the enemy’s bullet so it will head directly towards the player.
- In the game, Ryan used different txt files to record down the information about each level and room, so when creating the levels, information does not need to be created with in the codes, they can be simply read and translate into codes.
- Aaron built a fully functional back end that allowed the user to signup, login and logout their account. Also, Aaron managed the website through an admin page (Framework Featured).
- The game is downloadable through the website as a zip file, which showed my consideration to the users. Also, the Support and How To Play pages improve the user experience as well.

- When Peter and Ryan were putting the game together, they did not have lots of communication, which resulted in that the game took a lot of time to be put together, and there isn’t a lot of time to work on the graphics.
